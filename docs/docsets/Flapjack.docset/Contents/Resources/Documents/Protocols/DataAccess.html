DataAccess

public protocol DataAccess

One of two prominently-used objects in Flapjack, the DataAccess protocol (and those that conform to it) preside over the setup and management of the entire data persistence stack, along with managing the lifecycle of background context operations.

  • mainContext

    The main thread (or view-layer) context. Should generally only be used for read-only operations.

  • prepareStack(asynchronously:completion:)

    Invoking this method should ask the DataAccess object to load up its store from disk (or in memory), ask for any migrations needed, populate the necessary instance variables for accessing it, and set the isStackReady property to true if everything succeeded.

    Declaration

    Swift

    func prepareStack(asynchronously: Bool, completion: @escaping (DataAccessError?) -> Void)

    Parameters

    asynchronously

    If true, the stack preparation should be performed in a background thread, and the completion block should return on the main thread.
    completion

    A closure to be called upon completion. If asynchronously is true, this should be guaranteed to be called on the main thread. If false, it should be called on the calling thread.
